Migrate billing webhooks to Stripe v3
8 turns · started 14:32 · 28 Apr
Effort · medium 2 contexts
You 01

I need to migrate our Stripe webhook handler from v2 to v3. The main blocker is that charge.succeeded got renamed and the signature verification helper changed. Can you scan billing/webhooks.ts and outline the smallest possible diff?

02 Claude

Read the file. Two changes are doing the heavy lifting; everything else is a rename.

  • Event name. charge.succeeded → payment_intent.succeeded, with a slightly different payload shape (the charges.data[0] nesting is gone).
  • Signature. stripe.webhooks.constructEvent now takes the raw body buffer rather than the parsed string. That means express.raw() needs to be mounted before express.json() on this route.

Suggested patch:

// billing/webhooks.ts
- app.post('/stripe', express.json(), async (req, res) => {
-   const event = stripe.webhooks.constructEvent(
-     JSON.stringify(req.body),
-     req.header('stripe-signature') ?? '',
-     SIGNING_SECRET,
-   )
+ app.post('/stripe', express.raw({ type: 'application/json' }), async (req, res) => {
+   const event = stripe.webhooks.constructEvent(
+     req.body,
+     req.header('stripe-signature') ?? '',
+     SIGNING_SECRET,
+   )

Want me to apply it and run the test suite?
